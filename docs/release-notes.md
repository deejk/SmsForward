# 📋 릴리스 노트

SmsForward의 버전별 변경 내역.

---

## v1.0 (2026-06-02)

> versionCode `1` · versionName `1.0` · minSdk 26 (Android 8.0) / targetSdk 34
> 빌드: `build-release.bat` (release keystore 서명) · 배포: 갤럭시 S25 사이드로드

카드 결제 SMS를 지정 번호로 자동 전달하는 첫 정식 릴리스. 외부 서버·라이브러리 없이 **Android 기본 API만** 사용하며, 모든 처리는 온디바이스에서 이루어진다.

### ✨ 기능

- **자동 감지·전달**: SMS 수신 시 `BroadcastReceiver`가 즉시 카드 문자를 선별해 지정 번호로 재전송
- **정규식 매칭**: 삼성·신한카드 전용 패턴으로 승인/승인거절 문자를 정확히 식별 (택배·광고 등 오탐 차단)
- **📊 월 누적 요약**: 카드별 당월 누적 사용액을 자체 집계해 전달 문자 하단에 첨부
  ```
  📊 6월 누적 사용
  삼성: 1,250,000원
  신한: 340,000원
  ```
- **부팅 후 자동 시작**: 폰 재시작 시 `BootReceiver`가 서비스 자동 복구
- **Foreground Service**: 상시 알림으로 백그라운드 종료 방지
- **간단한 설정 UI**: 전달 번호·서비스 ON/OFF, 다크 모드 가독성 대응

### 🔐 서명·데이터

- **release keystore 서명 적용**: 재설치해도 SharedPreferences(카드 월 누적·설정)가 보존됨
- **본문 미보관**: 전달 문자 내용·이력은 저장하지 않음. 카드별 누적 **합계 숫자**만 보관 (달력 월 변경 시 자동 리셋)

### ⚠️ 알려진 한계

- **누적은 추정치**: 앱이 받은 승인 건만 합산 — 서비스 중단 중 결제·결제 취소는 미반영. 카드사 공식 청구액과 다를 수 있음 (SMS 내 `누적` 표기는 월 합계가 아니라 미사용)
- **삼성/신한 전용**: 그 외 카드사는 매칭 안 됨 (`SmsReceiver.kt`에 패턴 추가 필요)
- **요금 발생**: 전달 SMS마다 통신사 요금 (건당 약 22~33원)
- **민감 정보 전송**: 카드 결제 내역이 수신자에게 노출 — 신뢰 관계 전용
- **재시도 없음 / 듀얼 SIM 미지원 / 에뮬레이터 불가**

### 📦 설치 메모

- 디버그 서명 → release 서명 전환 시 **1회 삭제 후 재설치** 필요 (서명 불일치). 이후 release끼리는 데이터 유지.
- ⚠️ **keystore 백업 필수** — `smsforward-release.keystore` + `keystore.properties`는 git에 없음(공개 저장소 보호). 분실 시 같은 서명 재생성 불가. 상세: [`handover.md` §7](handover.md#7-️-release-keystore-백업-가장-중요)
